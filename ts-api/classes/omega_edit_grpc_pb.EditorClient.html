    Class EditorClient

    EditorService is the primary gRPC service for the Ωedit library. It provides byte-level editing of arbitrarily large files through a session/viewport model with unlimited, serialised undo/redo.

    Core concepts

    • Session – An editing context backed by an optional file. All edits (insert, delete, overwrite) are applied to a session. Sessions support checkpointing, transactions, and save-to-disk.

    • Viewport – A sliding window into a session's computed content. Viewports can be floating (automatically follow edits) or fixed (pinned to a byte offset). Multiple viewports may observe the same session.

    • Change – A single atomic edit (insert, delete, or overwrite). Every change receives a monotonically increasing serial number and can be undone or redone.

    • Event stream – Server-push notifications for session and viewport state changes, delivered via gRPC server-streaming RPCs.

# Ωedit™ Protocol Buffer Definitions

This directory contains the gRPC service definition for Ωedit™.

## `omega_edit/v1/omega_edit.proto`

Defines the `Editor` service with RPCs for:

- **Session management** — create, save, destroy editing sessions
- **Editing** — insert, delete, overwrite with unlimited undo/redo
- **Viewports** — sliding windows into session data
- **Search & profiling** — byte-pattern search, byte-frequency histograms
- **Event streams** — server-push notifications for session and viewport changes
- **Server lifecycle** — heartbeat, graceful/immediate shutdown

### Server Health Schema Note

The v1 server-health messages intentionally model the native C++ server now:

- `GetServerInfoResponse` uses `runtime_kind`, `runtime_name`, `platform`, `compiler`, `build_type`, and `cpp_standard`
- `GetHeartbeatResponse` uses optional process-memory fields:
  `resident_memory_bytes`, `virtual_memory_bytes`, and `peak_resident_memory_bytes`

Consumers migrating from older JVM-oriented fields should treat missing optional
heartbeat metrics as "unavailable" rather than `0`.
`virtual_memory_bytes` is best-effort and may be unset on platforms where an
equivalent process metric is not consistently available.

## Using with Buf

The proto is published to [buf.build/ctc-oss/omega-edit](https://buf.build/ctc-oss/omega-edit) for easy multi-language stub generation.

### Install the Buf CLI

```bash
# macOS / Linux
brew install bufbuild/buf/buf

# npm (cross-platform)
npm install -g @bufbuild/buf
```

### Generate stubs

From the repository root:

```bash
buf generate
```

This produces Go and Python stubs under `gen/`. See `buf.gen.yaml` to add more languages.

### Use as a dependency

In your own `buf.yaml`:

```yaml
version: v2
deps:
  - buf.build/ctc-oss/omega-edit
```

Then import directly in your `.proto` files:

```protobuf
import "omega_edit/v1/omega_edit.proto";
```

### Lint and breaking-change detection

```bash
buf lint
buf breaking --against '.git#ref=origin/main,subdir=proto'
```

## Generating stubs manually (without Buf)

### Go

```bash
protoc \
  --go_out=gen/go --go_opt=paths=source_relative \
  --go-grpc_out=gen/go --go-grpc_opt=paths=source_relative \
  -I proto \
  proto/omega_edit/v1/omega_edit.proto
```

### Python

```bash
python -m grpc_tools.protoc \
  --python_out=gen/python \
  --grpc_python_out=gen/python \
  -I proto \
  proto/omega_edit/v1/omega_edit.proto
```

### TypeScript (already generated by the client package)

The `@omega-edit/client` package generates TypeScript stubs as part of its build.
See `packages/client/DEVELOPMENT.md` for details.
